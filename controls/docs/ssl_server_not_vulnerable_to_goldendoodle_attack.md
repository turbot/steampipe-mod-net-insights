## Description

In **GOLDENDOODLE**, the attacker can systematically test all possible bytes until the correct byte is discovered (or all others are ruled out). GOLDENDOODLE can be used to hijack authenticated TLS sessions if the server reveals the padding validity of application data records so that a MiTM attacker can recognize well-formed padding independently from a valid Message Authentication Code (MAC). A server using a TLS version lower than `TLS v1.3` with `CBC` cipher modes is vulnerable to GOLDENDOODLE attack since the inherent weakness in CBC allows attackers to act as a man-in-the-middle (MITM) and execute the remaining attacks.
